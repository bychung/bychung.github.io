# 수학 학습 앱 MVP 기획서

## 1. 개요

본 문서는 태블릿 기반의 수학 학습 앱 'Math Grow (가칭)'의 MVP(Minimum Viable Product) 기획을 담고 있다. 중하위권 고등학생을 주요 타겟으로, 문제 풀이 과정에 대한 AI 피드백과 게이미피케이션 요소를 통해 학습 동기를 부여하고 실질적인 실력 향상을 유도하는 것을 목표로 한다.

## 2. MVP 핵심 목표

- 학생이 태블릿으로 손글씨 풀이 과정을 제출했을 때, AI의 분석 피드백과 `레벨(노력)` 및 `PSR(실력)` 시각화가 학습 동기부여에 긍정적인 영향을 미치는가?
- AI의 '풀이 과정 분석' 기술적 가능성 및 효용성 검증.

## 3. 핵심 기능 (Core Features)

## 4. MVP에서 제외할 기능

- 단원별/유형별 티어 (PSR의 단원별 분석으로 대체)
- AI 문제 추천 시스템 (학생이 직접 선택)
- AI 오답노트 자동 생성 및 관리
- 소셜/커뮤니티 기능 (랭킹, 스터디 그룹, Q&A 게시판 등)
- 고급 UI/UX 설정 (펜 커스텀, 테마 등)
- 복잡한 개인화 통계 대시보드 (레벨/PSR만 간결하게 노출)

## 5. 핵심 시스템 상세 기획

### 5.1. AI 풀이 분석 및 PScore (Performance Score) 시스템

AI는 학생의 풀이 과정을 **3단계 프롬프트 체인**을 통해 분석하여 PScore를 분류하고 상세한 피드백을 생성한다. 이 PScore는 PSR 변동량 계산의 핵심 지표가 된다.

#### 5.1.1. PScore 분류 유형 및 기준

| AI 분류 유형        | PScore (기준) | 설명 (AI 판단)                                               |
| :------------------ | :------------ | :----------------------------------------------------------- |
| Case 1: 완벽        | `1.0`         | 정답, 최적/효율적 풀이.                                      |
| Case 2: 아차\! 실수 | `0.7`         | 오답(또는 정답), 풀이 과정/개념 적용 완벽, 사소한 계산 실수. |
| Case 3: 돌아온 길   | `0.6`         | 정답, 풀이 과정 비효율적/불필요하게 김.                      |
| Case 4: 개념 부족   | `0.3`         | 오답, 풀이 과정에서 핵심 개념 잘못 적용.                     |
| Case 5: 우연        | `0.2`         | 정답, 풀이 과정 엉망/논리적 비약 심함 (찍어서 맞힌 경우).    |
| Case 6: 백지 제출   | `0.0`         | 풀이 없음 / 완전 무관한 풀이, 오답.                          |

**상세 분류 기준:**

- **Case 1: 완벽 (1.0)**
  - 조건: 정답 ✓, 풀이 과정 논리적·완결적, 개념 적용 정확, 효율적 풀이, 계산 실수 없음
  - 예시: 이차방정식을 인수분해로 정확히 풀고, 모든 단계가 명확

- **Case 2: 아차! 실수 (0.7)**
  - 조건: 정답 또는 오답, 풀이 과정·개념 완벽, **단순 계산 실수** (부호 실수, 사칙연산 오류)
  - 예시: 인수분해는 완벽했으나 x = -2를 x = 2로 잘못 씀

- **Case 3: 돌아온 길 (0.6)**
  - 조건: 정답 ✓, 풀이 **비효율적** (불필요한 단계 많음), 개념은 맞지만 최적 방법 아님
  - 예시: 인수분해로 풀 수 있는 문제를 근의 공식으로 풀고 추가 계산 많음

- **Case 4: 개념 부족 (0.3)**
  - 조건: 오답 ✗, **핵심 개념 잘못 이해·적용**, 풀이 시도는 있으나 근본적 오류
  - 예시: 이차방정식에서 양변에 x를 약분 (x=0 해 소실)

- **Case 5: 우연 (0.2)**
  - 조건: 정답 ✓ (하지만 우연), 풀이 비논리적, 심각한 논리적 비약, 찍어서 맞힌 것으로 판단
  - 예시: 엉뚱한 계산을 했는데 우연히 답이 나옴

- **Case 6: 백지 제출 (0.0)**
  - 조건: 풀이 아예 없음, 완전히 무관한 풀이, 오답 + 풀이 의미 없음
  - 예시: 빈 종이, 낙서, 전혀 다른 문제 풀이

#### 5.1.2. 3단계 프롬프트 체인 전략

정확도를 최우선으로 하기 위해, AI 분석을 3단계로 나누어 순차 실행:

```
[문제 이미지 + 풀이 이미지 + 학생 답안 + 정답]
    ↓
┌─────────────────────────────────────┐
│ STEP 1: 이미지 → 구조화된 텍스트   │
│ (OCR + 풀이 단계 추출)              │
│ 예상 시간: 2-3초                    │
└─────────────────────────────────────┘
    ↓
[풀이 구조화 데이터]
    ↓
┌─────────────────────────────────────┐
│ STEP 2: 논리 분석 및 PScore 분류   │
│ (개념/계산/흐름 검증)               │
│ 예상 시간: 2-3초                    │
└─────────────────────────────────────┘
    ↓
[분석 리포트 + PScore]
    ↓
┌─────────────────────────────────────┐
│ STEP 3: 상세 피드백 생성            │
│ (교육적 피드백 + 학습 가이드)       │
│ 예상 시간: 2-3초                    │
└─────────────────────────────────────┘
    ↓
[최종 피드백 JSON]
```

**총 예상 시간**: 6-9초 (병렬 처리 최적화 시 5-7초까지 단축 가능)

**장점:**
- 각 단계가 단일 목적에 집중 → 정확도 향상 (88-93%)
- 이미지 품질 문제 조기 감지 가능
- 디버깅 및 프롬프트 개선 용이
- 중간 결과 로깅으로 지속적 개선 가능

#### 5.1.3. STEP 1: 이미지 → 구조화된 텍스트

**목적**: 손글씨 이미지를 읽고 사실만 추출, 구조화된 데이터 생성

**프롬프트 핵심 요소:**
- 역할: 수학 풀이 OCR 전문가
- 입력: 문제 이미지, 풀이 이미지
- 작업: 문제 파악, 풀이 단계 추출, 가독성 평가
- 주의사항: 학생 내용을 절대 수정하지 않고 있는 그대로 추출

**출력 JSON 구조:**
```json
{
  "problem_info": {
    "type": "이차방정식",
    "content": "문제 내용 텍스트",
    "required_concepts": ["인수분해", "이차방정식의 해"]
  },
  "solution_steps": [
    {"step": 1, "content": "학생이 적은 내용", "is_readable": true},
    {"step": 2, "content": "...", "is_readable": true}
  ],
  "readability": {
    "problem_image": 0.9,
    "solution_image": 0.8,
    "overall": 0.85
  },
  "has_work": true,
  "work_completeness": 0.8
}
```

**조기 실패 처리**: readability < 0.5이면 중단하고 "이미지를 더 선명하게 찍어주세요" 안내

#### 5.1.4. STEP 2: 논리 분석 및 PScore 분류

**목적**: STEP 1의 텍스트를 바탕으로 논리적 타당성 검증 및 PScore 분류

**프롬프트 핵심 요소:**
- 역할: 수학 풀이 논리 검증 전문가
- 입력: STEP 1 JSON + 학생 답안 + 정답
- 분석 작업:
  1. 개념 사용 분석 (사용 개념, 적용 정확성)
  2. 계산 정확성 분석 (오류 위치, 유형, 원인)
  3. 논리적 연결성 분석 (흐름, 비약, 완결성)
  4. 효율성 분석 (최적 방법 여부)
  5. **PScore 결정 트리에 따른 분류**

**PScore 결정 트리:**
```
has_work == false OR work_completeness < 0.1?
├─ YES → Case 6 (백지, 0.0)
└─ NO
   └─ 최종 답안 정답?
      ├─ YES
      │  └─ 개념 적용 올바름?
      │     ├─ YES
      │     │  └─ 풀이 효율적?
      │     │     ├─ YES → Case 1 (완벽, 1.0)
      │     │     └─ NO → Case 3 (돌아온 길, 0.6)
      │     └─ NO → Case 5 (우연, 0.2)
      └─ NO
         └─ 개념 적용 올바름?
            ├─ YES
            │  └─ 단순 계산 실수만?
            │     └─ YES → Case 2 (아차 실수, 0.7)
            └─ NO → Case 4 (개념 부족, 0.3)
```

**출력 JSON 구조:**
```json
{
  "pscore_case": 2,
  "pscore_value": 0.7,
  "is_correct": false,
  "concept_analysis": {
    "used_concepts": ["인수분해"],
    "correct_concepts": ["인수분해"],
    "incorrect_concepts": [],
    "concept_score": 0.95
  },
  "calculation_analysis": {
    "errors": [
      {
        "step": 3,
        "type": "sign_error",
        "detail": "x - 3 = 0에서 x = -3으로 잘못 계산",
        "severity": "low"
      }
    ]
  },
  "logic_analysis": {
    "flow": "coherent",
    "completeness": 0.9
  },
  "key_strengths": ["인수분해 정확", "단계 명확"],
  "key_weaknesses": ["부호 실수"],
  "confidence": 0.95
}
```

#### 5.1.5. STEP 3: 상세 피드백 생성

**목적**: STEP 2 분석을 바탕으로 교육적이고 상세한 피드백 생성

**프롬프트 핵심 요소:**
- 역할: 중하위권 고등학생을 위한 따뜻하고 전문적인 수학 튜터
- 입력: STEP 2 JSON
- 피드백 원칙:
  - **톤**: 친근하고 격려적 ("~해요" 말투)
  - **구조**: 전체 요약 → 잘한 점 → 개선할 점 → 개념 설명 → 올바른 풀이 → 다음 단계
  - **길이**: 요약 30자, 잘한 점 각 80자, 개선점 각 150자, 개념 설명 200자
  - **긍정 우선**: 잘한 점을 먼저 언급 (Case 6 제외)

**출력 JSON 구조:**
```json
{
  "pscore_case": 2,
  "pscore_value": 0.7,
  "is_correct": false,
  "feedback": {
    "summary": {
      "emoji": "😊",
      "title": "아까운 실수!",
      "subtitle": "풀이는 완벽했는데, 마지막에 부호 실수가 있었어요"
    },
    "strengths": [
      {
        "title": "인수분해 정확",
        "detail": "x² - 5x + 6을 (x-2)(x-3)으로 완벽하게 인수분해했어요..."
      }
    ],
    "improvements": [
      {
        "title": "부호 실수",
        "what_went_wrong": "x - 3 = 0에서 x = -3으로 계산했어요",
        "why_wrong": "x - 3 = 0을 풀 때, 양변에 3을 더하면 x = 3...",
        "how_to_fix": "x - a = 0 형태에서는 x = a가 됩니다",
        "severity": "low"
      }
    ],
    "concept_explanation": {
      "concept_name": "이차방정식의 해 구하기",
      "explanation": "인수분해로 (x-a)(x-b) = 0 형태가 되면...",
      "example": "예를 들어, (x-5)(x+2)=0이면..."
    },
    "correct_solution": {
      "title": "올바른 풀이 과정",
      "steps": [
        {"step": 1, "content": "x² - 5x + 6 = 0", "explanation": "주어진 식"},
        {"step": 2, "content": "(x - 2)(x - 3) = 0", "explanation": "인수분해"}
      ]
    },
    "next_steps": {
      "advice": "부호 실수를 줄이려면, 풀이 과정을 한 번 더 검토하세요",
      "practice_topics": ["이차방정식의 해", "인수분해 확인"],
      "difficulty_adjust": "similar"
    }
  }
}
```

**PScore별 피드백 전략:**
- **Case 1 (완벽)**: 축하 + 더 어려운 문제 추천
- **Case 2 (아차 실수)**: 격려 + 실수 방지 팁
- **Case 3 (돌아온 길)**: 인정 + 더 효율적인 방법 소개
- **Case 4 (개념 부족)**: 개념 재설명 + 기초 복습 권장
- **Case 5 (우연)**: 논리적 풀이 중요성 강조 + 체계적 풀이 안내
- **Case 6 (백지)**: 따뜻한 격려 + 쉬운 문제부터 시작 권장

#### 5.1.6. UX 처리 전략

**단계별 로딩 메시지:**
- STEP 1 (0-3초): "📖 풀이를 꼼꼼히 읽고 있어요..."
- STEP 2 (3-6초): "🧮 개념과 계산을 확인 중이에요..."
- STEP 3 (6-9초): "💬 맞춤 피드백을 준비하고 있어요..."

**병렬 처리 최적화**: STEP 1에서 문제/풀이 이미지를 병렬로 처리하여 시간 단축 (2-3초 → 1.5-2초)

**중간 결과 로깅**: 각 단계 결과를 DB에 저장하여 나중에 분석 및 프롬프트 개선에 활용

### 5.2. PSR (Proficiency Skill Rating) 시스템 (학생 & 문제)

PSR은 학생과 문제 모두에게 부여되며, 풀이 결과에 따라 실시간으로 변동한다. 초기 PSR은 모두 1000점.

- **PSR 변동 공식:** `Rn = Ro + K * (PScore - E)`

  - `Rn`: 새로운 PSR, `Ro`: 현재 PSR
  - `K`: 변동 계수 (초기 높게, 안정화되면 낮게)
  - `PScore`: AI가 분석한 풀이의 '성과 점수' (위 표 참조)
  - `E`: 예상 결과 (`1 / (1 + 10^((상대방_PSR - 나의_PSR) / 400))`)

- **PScore에 '풀이 시간' 반영:**

  - **극초기 (문제별 풀이 데이터 N \< 10):**
    - 데이터를 모으는 시기로, PScore 조정 없음.
  - **초기 (문제별 풀이 데이터 10 ≤ N \< 30):**
    - 상위 10% 이내의 속도로 빠르게 푼 경우: PScore를 0.1~0.2 상향
    - 하위 10% 이내의 속도로 느리게 푼 경우: PScore를 0.1~0.2 하향
  - **중기 이후 (문제별 풀이 데이터 N ≥ 30):**
    - 해당 문제의 풀이 시간 `정규 분포`를 가정할 수 있는 상황이 됨.
    - 풀이 시간이 표준 편차 ±rσ 범위를 벗어나는 경우 PScore 조정.(0 < r < 1)
  - **시간 측정 기준:** '문제 풀이 시작'부터 '제출'까지의 시간. (정오답 여부와 무관)

- **K-Factor 동적 조절:**

  - **초기 문제:** 아직 PSR이 불안정한 문제에 K를 높게 설정하여 빠르게 수렴 유도.
  - **콜드 스타트:** N개 미만 데이터 문제에 K를 낮게 설정하여 초반의 부정확한 PSR 변동 방지.
    - N<10: K=20
    - 10<=N<30: K=30
    - 30<=N<100: K=40
    - N>100: K=20
  - **유저 PSR:** 유저의 PSR이 급변하거나 낮을 때 K를 높게 설정하여 빠르게 적정 실력을 찾아가도록 유도.

### 5.3. 레벨 시스템 (XP 기반)

레벨은 **노력의 지표**로, 학생의 학습 활동량과 꾸준함을 반영한다. PSR이 "실력"을 측정한다면, 레벨은 "얼마나 열심히 했는가"를 보여준다.

#### 5.3.1. 핵심 목표

- **하루 최소 5문제** 풀이 유도 (초반 일주일은 매일 레벨업 가능)
- **하루 이상적 10문제** 풀이 시 더 빠른 성장
- 레벨은 **절대 하락하지 않음** (심리적 안전망)

#### 5.3.2. XP 획득 체계

| 행동                        | XP        | 비고                                                      |
| :-------------------------- | :-------- | :-------------------------------------------------------- |
| **일일 첫 로그인**          | 50 XP     | 매일 접속 유도                                            |
| **문제 풀이 (기본)**        | 100 XP    | PSR 미고착 문제 (N<100)                                   |
| **문제 풀이 (PSR 기반)**    | PSR 차등  | PSR 고착 문제 (N≥100)<br>- PSR 1000~1100: 100 XP<br>- PSR 1100~1200: 120 XP<br>- PSR 1200~1300: 140 XP<br>- (100당 +20 XP) |
| **하루 3문제 달성**         | +100 XP   | 일일 퀘스트 1단계                                         |
| **하루 5문제 달성**         | +150 XP   | 일일 퀘스트 2단계 (최소 목표)                             |
| **하루 10문제 달성**        | +200 XP   | 일일 퀘스트 3단계 (이상적 목표)                           |
| **미개척 문제 (N≤10)**      | +300 XP   | 데이터 수집 인센티브 (기본 XP와 별도)                     |

**하루 XP 시뮬레이션:**

- **5문제 풀이**: 50 + 500 + 100 + 150 = **800 XP/일**
- **10문제 풀이**: 50 + 1,000 + 100 + 150 + 200 = **1,500 XP/일**

#### 5.3.3. 연속 달성 보너스 (스트릭)

꾸준한 학습을 장려하기 위한 연속 달성 보너스:

| 연속 달성               | 보너스 XP | 조건                  |
| :---------------------- | :-------- | :-------------------- |
| **7일 연속 5문제 이상** | +1,000 XP | 일주일 연속 달성      |
| **30일 연속 5문제 이상**| +5,000 XP | 한 달 연속 달성       |

- 연속이 끊기면 다시 1일차부터 시작.
- 스트릭 현황은 메인 대시보드에 "🔥 연속 N일째 도전 중!" 형태로 노출.

#### 5.3.4. 레벨업 공식

| 레벨 구간             | 레벨당 필요 XP           | 설명                       |
| :-------------------- | :----------------------- | :------------------------- |
| **Lv.1~7 (초반)**     | 고정 **750 XP**          | 매일 레벨업 가능 구간      |
| **Lv.8~15 (중반)**    | 900 + (n-8) × 150        | 점진적 증가 시작           |
| **Lv.16~30 (후반)**   | 2,000 + (n-16) × 200     | 의미있는 성취              |
| **Lv.31+ (최상위)**   | 5,000 + (n-31) × 300     | 장기 목표                  |

**예시:**

- Lv.1→2: 750 XP (하루 5문제로 ~1일)
- Lv.7→8: 750 XP (일주일차에 달성)
- Lv.10→11: 1,200 XP (하루 5문제로 ~1.5일)
- Lv.20→21: 2,800 XP (하루 5문제로 ~3.5일)

#### 5.3.5. 레벨 구간별 타이틀

레벨 달성 시 해당 구간의 타이틀이 부여되어 학습 여정의 의미를 더한다.

| 레벨 구간      | 타이틀                      | 설명                               |
| :------------- | :-------------------------- | :--------------------------------- |
| **Lv.1~7**     | 🌱 **새싹 계산사**          | 수학 여정을 시작한 탐험가          |
| **Lv.8~15**    | 🧮 **방정식 탐험가**        | 문제 해결의 재미를 알아가는 단계   |
| **Lv.16~30**   | 📐 **기하학 모험가**        | 다양한 유형에 도전하는 숙련자      |
| **Lv.31~50**   | 🎯 **함수 마스터**          | 체계적 풀이의 달인                 |
| **Lv.51~80**   | 🏆 **수학 올림피안**        | 꾸준함으로 쌓아올린 실력           |
| **Lv.81~120**  | ⚡ **오일러의 후예**        | 수학적 사고의 경지                 |
| **Lv.121~200** | 🌟 **가우스의 계승자**      | 전설의 문제 풀이 여정              |
| **Lv.201+**    | 👑 **무한의 탐구자**        | 끝없는 성장을 추구하는 자          |

- 타이틀은 프로필 및 메인 대시보드에 노출.
- 레벨업 시 새 타이틀 획득하면 축하 애니메이션 표시.

#### 5.3.6. 최대 레벨

- **최대 레벨 제한 없음** (사실상 무한).
- 장기적으로 Lv.1000 이상도 도달 가능하도록 설계.
- 매우 높은 레벨은 그 자체로 "전설적 헌신"의 상징.

- **잠금/획득 상태**: 회색(미획득) / 컬러(획득)으로 시각적 구분.
- **진행도 표시**: 달성 가능한 뱃지에 대해 프로그레스 바 제공 (예: "누적 50문제 달성까지 23/50").
- **알림**: 뱃지 획득 시 축하 팝업 + 애니메이션으로 성취감 강화.
- **뱃지 컬렉션 페이지**: 모든 뱃지를 한눈에 볼 수 있는 별도 페이지 제공.

### 5.5. UX/UI 상 지표 노출 전략

- **메인 대시보드:** 학생의 `레벨`과 `PSR` (총점)을 **간결하게** 노출. (긍정적 동기 부여)
  - `OOO님은 Level XX 입니다!`
  - `현재 수학 실력 점수: XXXX점`
- **'실력 인바디' 페이지 (별도 진입):** 상세 PSR 분석.
  - PSR 변동 그래프.
  - AI 코멘트 (PSR 상승/하락 요인 분석, 강점/약점 단원 언급).
  - 단원별 PSR (V1.0 기능이지만, MVP에서는 'AI 코멘트'에 간략히 언급 가능).
